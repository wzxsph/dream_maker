# NovelForge 作品卡片生成流程文档

## 1. 项目卡片生成层级总览

```
项目创建
    │
    ├── 作品标签 (singleton, 手动创建)
    │
    ├── 金手指 (singleton, AI生成)
    │       │
    │       └── 提示词模板: 金手指生成.txt
    │
    ├── 一句话梗概 (singleton, AI生成)
    │       │
    │       └── 提示词模板: 一句话梗概.txt
    │
    ├── 故事大纲 (singleton, AI生成)
    │       │
    │       └── 提示词模板: 一段话大纲.txt
    │
    ├── 世界观设定 (singleton, AI生成)
    │       │
    │       └── 提示词模板: 世界观设定.txt
    │
    ├── 核心蓝图 (singleton, AI生成)
    │       │
    │       ├── 提示词模板: 核心蓝图.txt
    │       └── 触发工作流: 核心蓝图.wf
    │              │
    │              ├──→ [N] 分卷大纲 (按 volume_count)
    │              ├──→ [M] 角色卡 (从 blueprint.content.character_cards)
    │              └──→ [K] 场景卡 (从 blueprint.content.scene_cards)
    │
    ├── 分卷大纲 (auto-created, AI生成)
    │       │
    │       ├── 提示词模板: 分卷大纲.txt
    │       └── 触发工作流: 分卷大纲.wf
    │              │
    │              ├──→ [P] 角色卡 (从 new_character_cards)
    │              ├──→ [Q] 场景卡 (从 new_scene_cards)
    │              ├──→ [S] 阶段大纲 (按 stage_count)
    │              └──→ 写作指南
    │
    ├── 阶段大纲 (auto-created, AI生成)
    │       │
    │       ├── 提示词模板: 阶段大纲.txt
    │       └── 触发工作流: 阶段大纲.wf
    │              │
    │              ├──→ [X] 章节大纲 (从 chapter_outline_list)
    │              └──→ [X] 章节正文 (并行创建)
    │
    ├── 章节大纲 (auto-created)
    │
    ├── 章节正文 (auto-created)
    │
    └── 写作指南 (auto-created)
```

## 2. 卡片类型定义详解

### 2.1 单例卡片 (Singleton)

| 卡片类型 | 是否单例 | AI上下文模板 | 使用提示词 |
|---------|---------|-------------|-----------|
| 作品标签 | ✓ | 无 (手动) | - |
| 金手指 | ✓ | 作品标签: @作品标签.content | 金手指生成.txt |
| 一句话梗概 | ✓ | 作品标签 + 金手指 | 一句话梗概.txt |
| 故事大纲 | ✓ | 作品标签 + 金手指 + 一句话梗概 | 一段话大纲.txt |
| 世界观设定 | ✓ | 作品标签 + 金手指 + 故事大纲 | 世界观设定.txt |
| 核心蓝图 | ✓ | 作品标签 + 金手指 + 故事大纲 + 世界观 + 组织卡 | 核心蓝图.txt |

### 2.2 自动创建卡片

| 源卡片 | 创建目标 | 触发方式 | 创建逻辑 |
|-------|---------|---------|---------|
| 核心蓝图 | 分卷大纲 | 工作流 | 按 `volume_count` 生成 N 卷 |
| 核心蓝图 | 角色卡 | 工作流 | 从 `character_cards` 列表提取 |
| 核心蓝图 | 场景卡 | 工作流 | 从 `scene_cards` 列表提取 |
| 分卷大纲 | 角色卡 | 工作流 | 从 `new_character_cards` 提取 |
| 分卷大纲 | 场景卡 | 工作流 | 从 `new_scene_cards` 提取 |
| 分卷大纲 | 阶段大纲 | 工作流 | 按 `stage_count` 生成 |
| 分卷大纲 | 写作指南 | 工作流 | 直接创建 |
| 阶段大纲 | 章节大纲 | 工作流 | 从 `chapter_outline_list` 提取 |
| 阶段大纲 | 章节正文 | 工作流 | 并行创建 |

## 3. AI 生成流程详解

### 3.1 单例卡片 AI 生成流程

```
用户触发AI生成
       │
       ▼
  调用 /api/ai/generate/stream
       │
       ▼
  构建系统提示词
  ├─ AI参数预设 (temperature, max_tokens, timeout)
  ├─ 卡片类型提示词模板 (如 核心蓝图.txt)
  └─ 指令流生成规范
       │
       ▼
  LLM 指令流输出
  ├─ type: "thinking" → AI自然语言思考
  ├─ type: "instruction" → JSON指令 {"op":"set","path":"/field","value":...}
  └─ type: "done" → 完成验证
       │
       ▼
  指令应用与验证
  ├─ validate_instruction() 验证指令
  └─ apply_instruction() 应用指令
       │
       ▼
  输出最终结构化数据
```

### 3.2 工作流自动创建流程

```
用户保存源卡片 (如 核心蓝图)
       │
       ▼
  触发器: Trigger.CardSaved(card_type="核心蓝图")
       │
       ▼
  读取卡片内容
       │
       ▼
  Logic.Expression 表达式计算
  例: [ {'index': i} for i in range(1, volume_count + 1) ]
       │
       ▼
  Card.BatchUpsert 批量创建
  ├─ project_id, card_type
  ├─ title_template (如 "第{item.index}卷")
  └─ content_template (如 {"volume_number": "{item.index}"})
       │
       ▼
  Logic.Wait 等待异步任务完成
       │
       ▼
  Card.Update 清空来源列表
  例: { "character_cards": [], "scene_cards": [] }
```

## 4. 工作流文件详解

### 4.1 核心蓝图.wf

```
触发器: Trigger.CardSaved(card_type="核心蓝图")

节点:
1. Card.Read → 读取核心蓝图卡片
2. Logic.Expression → 生成分卷列表
   expression = [ {'index': i} for i in range(1, blueprint.content.volume_count + 1) ]
3. Card.BatchUpsert (异步) → 批量创建分卷大纲
4. Logic.Expression → 提取角色列表
   expression = blueprint.content.character_cards or []
5. Card.BatchUpsert (异步) → 批量创建角色卡
6. Logic.Expression → 提取场景列表
   expression = blueprint.content.scene_cards or []
7. Card.BatchUpsert (异步) → 批量创建场景卡
8. Logic.Wait → 等待所有异步任务完成
9. Card.Update → 清空 character_cards 和 scene_cards
```

### 4.2 分卷大纲.wf

```
触发器: Trigger.CardSaved(card_type="分卷大纲")

节点:
1. Card.Read → 读取分卷大纲卡片
2. Logic.Expression → 提取新角色列表
3. Card.BatchUpsert (异步) → 批量创建角色卡
4. Logic.Expression → 提取新场景列表
5. Card.BatchUpsert (异步) → 批量创建场景卡
6. Logic.Expression → 生成阶段列表
   expression = [ {'index': i, 'volume_number': volume.content.volume_number}
                for i in range(1, stage_count + 1) ]
7. Card.BatchUpsert (异步) → 批量创建阶段大纲
8. Logic.Expression → 包装写作指南内容
9. Card.BatchUpsert (异步) → 创建写作指南
10. Logic.Wait → 等待所有异步任务完成
11. Card.Update → 清空 new_character_cards 和 new_scene_cards
```

### 4.3 阶段大纲.wf

```
触发器: Trigger.CardSaved(card_type="阶段大纲")

节点:
1. Card.Read → 读取阶段大纲卡片
2. Logic.Expression → 提取并增强章节大纲列表
   expression = [ {**outline, 'title': '第' + chapter_number + '章 ' + title,
                   'entity_list': outline.get('entity_list') or [],
                   'volume_number': stage.content.volume_number,
                   'stage_number': stage.content.stage_number}
                  for outline in stage.content.chapter_outline_list ]
3. Logic.Expression → 校验章节号连续性
4. Card.BatchUpsert (异步) → 批量创建章节大纲
5. Card.BatchUpsert (异步) → 批量创建章节正文
6. Logic.Wait → 等待异步任务完成
7. Card.Update → 清空 chapter_outline_list
```

## 6. AI 上下文模板 (Card Type)

### 6.1 金手指
```
作品标签: @作品标签.content
```

### 6.2 一句话梗概
```
作品标签: @作品标签.content
金手指/特殊能力: @金手指.content.special_abilities
```

### 6.3 故事大纲
```
作品标签: @作品标签.content
金手指/特殊能力: @金手指.content.special_abilities
故事梗概: @一句话梗概.content.one_sentence
```

### 6.4 世界观设定
```
作品标签: @作品标签.content
金手指/特殊能力: @金手指.content.special_abilities
故事大纲: @故事大纲.content.overview
```

### 6.5 核心蓝图
```
作品标签: @作品标签.content
金手指/特殊能力: @金手指.content.special_abilities
故事大纲: @故事大纲.content.overview
世界观设定: @世界观设定.content
组织/势力设定: @type:组织卡[previous:global].{content.name, content.description, content.influence, content.relationship}
```

### 6.6 分卷大纲
```
总卷数: @核心蓝图.content.volume_count
故事大纲: @故事大纲.content.overview
作品标签: @作品标签.content
世界观设定: @世界观设定.content.world_view
组织/势力设定: @type:组织卡[previous:global].{content.name, content.description, content.influence, content.relationship}
角色卡: @type:角色卡[previous]
场景卡: @type:场景卡[previous]
上一卷信息: @type:分卷大纲[index=$current.volumeNumber-1].content
接下来请你创作第 @self.content.volume_number 卷的细纲
```

### 6.7 阶段大纲
```
世界观设定: @世界观设定.content.world_view
组织/势力设定: @type:组织卡[previous:global].{...}
分卷主线: @parent.content.main_target
分卷辅线: @parent.content.branch_line
角色卡信息: @type:角色卡[previous:global].{...}
地图/场景卡信息: @type:场景卡[previous]
该卷的角色行动简述: @parent.content.character_action_list
之前的阶段故事大纲: @type:阶段大纲[previous:global:1].{...}
上一章节大纲概述: @type:章节大纲[previous:global:1].{content.overview}
本卷的StageCount总数为: @parent.content.stage_count
注意，请务必在@parent.content.stage_count 个阶段内将故事按分卷主线收束
该卷的写作注意事项: @type:写作指南[sibling].content.content
接下来请你创作第 @self.content.stage_number 阶段的故事细纲
```

### 6.8 章节正文
```
世界观设定: @世界观设定.content
组织/势力设定: @type:组织卡[index=filter:...]
场景卡: @type:场景卡[index=filter:...]
当前故事阶段大纲: @parent.content.overview
角色卡: @type:角色卡[index=filter:content.name in $self.content.entity_list].{...}
物品卡: @type:物品卡[index=filter:...]
概念卡: @type:概念卡[index=filter:...]
最近的章节原文: @type:章节正文[previous:1].{content.title, content.chapter_number, content.content}
参与者实体列表: @self.content.entity_list
请根据 @self.content.chapter_number: @self.content.title 的大纲来创作章节正文内容
注意，写作时必须保证结尾剧情与下一章的剧情大纲不会冲突
写作时请结合写作指南要求: @type:写作指南[index=filter:content.volume_number = $self.content.volume_number]
```

## 8. 总结

NovelForge 的卡片生成采用**层级触发式**架构：

1. **手动创建**: 作品标签为起点，需用户手动创建
2. **AI生成单例**: 金手指→一句话梗概→故事大纲→世界观设定→核心蓝图，依次依赖前置卡片内容
3. **工作流自动创建**: 核心蓝图保存触发工作流，自动生成分卷、角色卡、场景卡
4. **层级递进**: 每个分卷可生成多个阶段，阶段自动生成章节大纲和章节正文
5. **上下文注入**: 通过 `@DSL` 语法引用其他卡片内容，保证创作一致性
