# **产品立项手册：造梦**

## **一、 产品定位**

* **一句话总结**：一句话生成你的专属互动短剧。  
* **产品介绍**：「造梦」是一款基于大模型驱动的极简互动内容社区。用户仅需输入一句话或一个脑洞，即可瞬间生成带有分支选项的竖屏互动小说。结合抖音碎片化消费习惯，单局体验控制在3分钟内。产品提供“轻创作+快消费”的 UGC 闭环，创作者发布内容，玩家通过互动或观看激励视频解锁剧情。在极低的操作门槛下，为用户提供密集的爽点与情绪价值。

## **二、 核心运转流程（输入-处理-输出）**

### **1\. 线下完整版架构（真机 AI 驱动模式）**

* **输入**：用户通过输入框自由输入一段开放式 Prompt（如“我重生成了一只猫，并且带了系统”），以及在节点处点击“A/B”选项。  
* **处理过程**：前端捕获输入并将其封装进预设的 System Prompt 模板中，发起网络请求调用大模型 API。大模型按照严格的 JSON Schema 返回当前剧情文本、场景氛围标签、以及两个后续动作选项。后台维护上下文（Context）记忆，确保状态连续性。  
* **输出**：前端通过解析 JSON，动态更新 DOM 树。基于返回的氛围标签，利用 CSS3 渐变过渡、关键帧动画（如屏幕震动、闪烁）和本地微型 SVG 图标实时渲染页面UI，完成动态视觉反馈。

### **2\. 线上 Demo 版架构（离线降级伪装模式）**

* **输入**：用户在输入框输入文本（前端底层做模糊关键词提取或提供伪造的“热门搜索词”供点击），以及在节点处点击选项。  
* **处理过程**：绝对物理断网。前端拦截用户的输入，通过正则表达式匹配预埋在本地的多个高通用性 JSON 决策树（例如“修仙流”、“霸总流”、“惊悚流”等剧本树）。匹配不到时触发兜底的“万能穿越”剧本。用户点击选项时，本地 JS 引擎作为状态机（State Machine），根据当前节点的 next\_node\_id 在 JSON 树中进行 O(1) 复杂度的本地路由寻址。  
* **输出**：复用完整版的 CSS3 和 Canvas 渲染逻辑，所有文本、分支与 UI 动效均从本地 JSON 中读取并渲染。全量代码与 JSON 数据压缩后控制在 2MB 内，完全规避网络请求，符合 8MB 包体和无网审核红线。

## **三、 设计哲学**

* **低认知负荷的“滑/点”本能交互**：摒弃传统 AVG 游戏复杂的背包、属性与存档系统。全局界面设计对标短视频信息流，采用极简的大色块、大字体与竖屏单手操作。用户只需进行“二选一”的点按，实现“开箱即玩，本能决策”，将学习成本降至绝对零。  
* **高频反转与即时情绪反馈**：剧本节点设计严格遵循“爽文”逻辑，每 3-5 次点击操作必出现一次剧情高潮、系统奖励或生死反转。配合 CSS3 原生动效（如选择正确时的礼花粒子、遭遇危机时的屏幕血红闪跳），提供极短链路的多巴胺视觉与心理双重刺激。  
* **悬念截断驱动的社交裂变**：利用抖音的传播生态，在剧情推演至最高潮的“生死抉择”或“揭秘”节点时设立机制锁。玩家需要将当前剧情卡片分享至抖音/好友，或点赞原作者，方可解锁核心分支或隐藏结局，将内容消费转化为高转化的社交货币。

## **四、 商业与生态闭环 (MVP)**

* **内容供给 (生产)**：将“写小说”降维成“抛梗”。提供“热门脑洞生成器”（如给出几个 Tag 组合），用户一键确认即可由系统生成完整分支树（线上真机版），或由官方预先产出优质 UGC 种子库（Demo版），实现零门槛内容冷启动。  
* **流量分发 (消费)**：建立极简的推荐漏斗。MVP 阶段不依赖复杂推荐算法，仅通过“单局通关率（完播率）”与“分支点击率（互动率）”双指标赛马。数据达标的剧情卡片自动流入公域池，以类似图文卡片的形式在社区瀑布流中分发。  
* **变现机制 (商业)**：采用“情绪付费”节点前置策略。在关键的“逆袭/复仇”分支，或玩家角色死亡需要“时间回溯”时，弹出激励视频广告（线上版对接穿山甲等 SDK，离线 Demo 版用本地 UI 弹窗伪装演示逻辑）。跑通“用户消费-看广解锁-平台变现-作者分润”的极简商业模型。

## **五、 产品具体案例演示：以《真千金重生：踢碎反派光环》为例**

* **1\. 用户输入脑洞**：用户在首页输入框输入：“我重生回到了假千金把我推下楼梯的那天。”并点击生成。  
* **2\. 初始场景 (Node\_0)**：  
  * **剧情文本**：一阵剧烈的眩晕后，你猛地睁开眼，发现自己正站在别墅二楼的楼梯边缘。面前是假千金林婉儿楚楚可怜的脸，她正伸手向你推来，嘴角挂着冷笑，嘴里却惊呼着：“姐姐，别怪我……”  
  * **UI/动效反馈**：屏幕边缘闪烁微弱白光（重生特效），手机触发 0.5 秒轻微震动（心跳悸动感），背景音效切入急促的弦乐。  
  * **选项分支**：  
    * \[选项 A\] 惊慌失措，侧身闪躲，顺势拉她一把。  
    * \[选项 B\] 重生不憋屈！反手就是一个清脆的巴掌，大喊“你发什么疯！”  
* **3\. 核心抉择与反转 (Node\_1)**：  
  * **\[用户点击 选项 B\]**  
  * **剧情文本**：“啪！”清脆的巴掌声响彻大厅！林婉儿捂着高肿的脸跌倒在地，满眼不可置信。就在这时，你的豪门未婚夫男主刚好推门进来，看到这一幕，怒斥道：“你在干什么？！婉儿身体这么弱，你还要打她？”  
  * **UI/动效反馈**：屏幕伴随“啪”的音效产生剧烈震动（打击反馈），UI 主色调瞬间变暗变冷（压迫感），视觉焦点聚集在男主的斥责文本上。  
  * **选项分支**：  
    * \[选项 A\] 瞬间落泪，装得比她更柔弱委屈：“是她先动手的……”  
    * \[选项 B\] 冷笑一声，指着男主鼻子：“眼瞎就去治，你也配管我？”  
* **4\. 商业化拦截节点 (Node\_X)**：  
  * **\[用户点击 选项 B\]**  
  * **剧情文本**：男主被你怼得愣在原地。突然，你的脑海中响起刺耳的机械音：【警告！检测到宿主严重偏离“受气包”剧情线，世界意志即将施加致命惩罚！倒计时 3...2...1...】  
  * **UI/动效反馈**：全屏闪烁高频红光，心跳音效急促倒数，UI 界面产生类似“系统崩溃”的代码乱码特效。倒计时至“1”时，所有动画静止，画面被一层半透明黑底遮罩锁定。  
  * **拦截机制**：屏幕中央弹出高亮卡片——“【致命危机】当前剧情已被世界意志锁定！立即观看视频，获取道具「系统防火墙」进行反杀！”（包含“看广告解锁”按钮与“转发求助”按钮）。

## **六、 互动小说标准架构范式 (JSON Schema)**

* **核心设计理念**：采用基于字典的无向图（State Machine）模型，实现 O(1) 的节点寻址。摒弃数值计算，实现剧情逻辑与前端渲染的完全解耦。  
* **标准 Schema 示例**：

{  
  "story\_id": "dream\_rebirth\_001",  
  "title": "真千金重生",  
  "nodes": {  
    "node\_0": {  
      "text": "一阵剧烈的眩晕后，你猛地睁开眼，发现自己正站在别墅二楼的楼梯边缘...",  
      "ui\_effect": \["flash\_white", "vibrate\_light"\],  
      "bg\_theme": "light",  
      "choices": \[  
        {  
          "content": "侧身闪躲",  
          "next\_node": "node\_1\_a"  
        },  
        {  
          "content": "反手一巴掌",  
          "next\_node": "node\_1\_b"  
        }  
      \]  
    },  
    "node\_1\_b": {  
      "text": "机械音响起：【警告！检测到宿主严重偏离剧情线...】",  
      "ui\_effect": \["flash\_red", "glitch", "vibrate\_heavy"\],  
      "bg\_theme": "dark",  
      "is\_paywall": true,  
      "paywall\_type": "ad\_video",  
      "choices": \[  
        {  
          "content": "使用「系统防火墙」反杀",  
          "next\_node": "node\_2\_success"  
        }  
      \]  
    }  
  }  
}

* **核心字段解析**：

| 字段名 | 数据类型 | 是否必填 | 作用与说明 |
| :---- | :---- | :---- | :---- |
| story\_id | String | 是 | 故事模型的全局唯一标识符。 |
| nodes | Object | 是 | 节点集合字典（状态机的核心 Map），Key 为 node\_id。 |
| \[node\_id\] | String | 是 | 节点的唯一标识（如 node\_0, node\_1\_a），用于路由跳转寻址。 |
| text | String | 是 | 当前节点的主剧情文案，直接用于前端 DOM 渲染。 |
| ui\_effect | Array\[String\] | 否 | 前端视觉/触觉触发标签，如 flash\_red（红闪）、vibrate（震动）。 |
| bg\_theme | String | 否 | 背景氛围主题，控制全局 CSS 色调（如 light, dark, bloody）。 |
| choices | Array\[Object\] | 是 | 分支选项列表，严格限制最多包含 2 个对象（保障二选一低认知）。 |
| choices\[\].content | String | 是 | 展示在前端按钮上的选项文案。 |
| choices\[\].next\_node | String | 是 | 选定该项后，状态机下一步要跳转的 node\_id。 |
| is\_paywall | Boolean | 否 | 标记当前节点是否为商业化卡点（默认为 false）。 |
| paywall\_type | String | 否 | 变现方式（如 ad\_video 激励视频、share 社交裂变）。 |
