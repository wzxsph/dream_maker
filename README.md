# 造梦 (Dream Maker)

> 一句话生成你的专属互动短剧

## 产品概述

🚧 **待开发产品** - 目前处于产品设计 & MVP 验证阶段

「造梦」是一款基于大模型驱动的极简互动内容社区。用户仅需输入一句话或一个脑洞，即可瞬间生成带有分支选项的竖屏互动小说。提供"轻创作+快消费"的 UGC 闭环，在极低的操作门槛下，为用户提供密集的爽点与情绪价值。

项目生成的故事统一服从 [微场景锁结局三幕架构](./STORY_ARCHITECTURE.md)：用小场景、少角色、伪开放分支和锁定结局走向，换取更高生成效率和更稳定的互动体验。

## 核心特性

- **零门槛创作**：输入一句话脑洞，AI 自动生成三幕短篇互动小说
- **微场景架构**：单一核心场景 + 最多 3 个有姓名角色，减少发散和跑题
- **伪开放分支**：选择改变手段、代价和信息揭露顺序，但故事在每幕内汇聚
- **锁定结局走向**：从真相翻盘、代价脱身、身份换位三类结局中选一并贯穿全文
- **本能级交互**：极简"点/滑"操作，摒弃复杂背包、属性与存档系统
- **高频情绪反转**：每个节点提供可见压力、信息差或强情绪推进
- **悬念截断裂变**：在最高潮节点设立机制锁，将内容消费转化为社交货币
- **包体极简**：严格控制在 ≤8MB

## 技术架构

- **前端**：纯静态 HTML/CSS/JS，无框架依赖
- **状态机**：基于 JSON Schema 的 O(1) 路由寻址节点系统
- **叙事约束**：`backend/config/storyArchitecture.js` 统一约束节点图、场景锁与结局走向
- **降级策略**：支持线上 API 与离线本地模式智能切换

## 商业闭环

1. **内容供给 (生产)**：热门脑洞生成器，零门槛冷启动
2. **流量分发 (消费)**：基于"通关率+分支点击率"赛马机制
3. **变现机制 (商业)**：情绪付费节点前置，激励视频广告拦截
